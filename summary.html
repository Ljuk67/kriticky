📋 Project Overview

Kriticky.sk is a Slovak-language educational blog focused on critical thinking and media literacy. The project aims to help ordinary people develop critical thinking skills to navigate the modern information landscape, identify misinformation, and make informed decisions.

"Informácia nie je to isté ako pravda. Väčšina informácií nie je presným zobrazením reality."
— Yuval Noah Harari

Mission & Goals

  • Strengthen rational thinking and critical analysis among regular people
  • Explain logical fallacies and manipulative techniques
  • Show how to verify claims and evaluate sources
  • Provide practical guides in accessible language
  • Help people think more critically, calmly, and systematically

Target Audience

People who want to better understand information around them and make calmer, more informed decisions. Specifically targeting those who:

  • Feel something doesn't add up but can't explain why
  • Are naturally curious but cautious
  • Don't believe everyone can have "their own truth"
  • Want to understand the world without being manipulated

🛑 The STOP Method

A simple framework for evaluating claims and information:

S

Skontroluj zdroj

Check the source - Who is making this claim? Is it trustworthy? Do they have expertise or a history of dishonesty?

T

Triezvo premýšľaj

Think soberly - Is it playing on your emotions? Does it sound too good/bad to be true?

O

Over fakty

Verify facts - Can claims be confirmed from multiple independent sources? Is there evidence, studies, real data?

P

Pozor na zaujatosť

Watch for bias - Your own: Are you only seeking confirmation? Author's: What's their motivation?

✨ Key Features

1. Content Management

  • Blog posts in Markdown/MDX with frontmatter metadata
  • Draft/published status system (drafts hidden in production)
  • Tags and categories for content organization
  • Automatic RSS feed generation
  • Sitemap integration for SEO

2. Footnotes System

  • Inline footnotes with popup explanations for technical terms
  • Shared term dictionary in public/footnotes-terms.json
  • Auto-annotation across the entire site
  • Helps make complex concepts accessible to general audience

3. Comment System

  • Email verification required before comments appear
  • Supabase backend with RLS (Row Level Security)
  • PHP endpoints for submission and verification
  • Admin notifications when comments are verified
  • SMTP integration via Websupport

4. Search Functionality

  • Client-side search powered by Pagefind
  • Built during the build process
  • Fast, privacy-friendly (no external services)

5. Automated Thumbnail Generation

  • SVG thumbnails auto-generated for blog posts
  • Abstract geometric patterns with gradients
  • Regenerated when content changes
  • Optimized for performance and visual appeal

6. Link Checking

  • Automatic link validation after each build
  • Checks internal links and optionally external links
  • Anchor verification support
  • Reminder system if not run in 30+ days

7. Content Linking Rules

  • Automatic cross-linking of key concepts (e.g., "spoľahlivé zdroje")
  • Links to foundational articles on confirmation bias, source evaluation
  • Smart self-link prevention

🚀 Development Workflow

Commands

  • nvm use - Switch to Node.js 20
  • npm ci - Clean install dependencies
  • npm run dev - Start dev server with hot reload
  • npm run build - Create production build
  • npm run preview - Preview production build locally
  • npm run check:site - Check for broken links
  • npm run gen:thumbs - Generate thumbnails manually

Branching Strategy

  • Never commit directly to master
  • Create feature branches: feat/<scope>
  • Create content branches: content/<slug>
  • Create fix branches: fix/<scope>
  • Open pull requests for review before merging

Content Guidelines

  • Use informal Slovak (ty/teba, not Vy)
  • Target audience: people unfamiliar with critical thinking
  • Explain technical terms in brackets or footnotes
  • Use callouts for examples, tips, and takeaways
  • One actionable takeaway per section
  • Concrete examples over abstract theory

🌐 Hosting & Deployment

  • Hosting: Websupport (Slovak hosting provider)
  • Domain: kriticky.sk
  • Deployment: Build locally or via GitHub Actions, upload dist/ to web root
  • Analytics: Privacy-friendly, GDPR-compliant solution preferred
  • No CDN: Not using Cloudflare or Netlify

🎯 Design Philosophy

  • Transparency: Always cite sources and provide context
  • Clarity: Plain language and practical examples
  • Fairness: Respect for people with different opinions
  • Slowness over speed: Better to verify than be first to share
  • Accessibility: Make complex concepts understandable to everyone
  • Non-partisan: Not a political campaign or "fight against others"
